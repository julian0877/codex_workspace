# 钢结构技术标书 Word 自动格式化程序

这是一款 Windows 桌面工具，用于识别钢结构技术标书中的一级至三级标题、正文、表格、图片和题注，并根据企业 Word 样板及本项目规则生成新的规范化文档。程序不会覆盖源文件。

## 运行条件

- Windows 10 或 Windows 11；
- Microsoft Word 桌面版；
- 源文件和样板均为未受保护的 `.docx`；
- 源码运行需要 Python 3.12。

## 样板要求

样板必须配置以下 Word 样式：

- `Normal`：正文；
- `Heading 1`、`Heading 2`、`Heading 3`：一级至三级标题；
- `Caption`：图片题注。

封面中需要自动替换的值使用显式占位符，例如：

```text
{{项目名称}}
{{投标单位}}
{{日期}}
{{目录}}
```

`{{目录}}` 是自动目录的插入位置；其他占位符会在第二步自动显示为必填输入框。程序只替换显式占位符，不猜测普通封面文字。

## 源码启动

```powershell
python -m pip install -e ".[dev]"
python -m tender_formatter.main
```

## 使用流程

1. 选择待处理技术标书、企业 Word 样板和输出文件。
2. 检查从样板读取的规则，并修改本项目的字号、页边距、图片宽度等要求。
3. 检查程序标出的复核项和高风险项，确认标题类型与层级。
4. 生成新文档。结果页显示输出 `.docx` 和脱敏检查报告的位置。

默认输出名为 `<原文件名>_已格式化.docx`。报告只记录段落数量、操作数量和警告，不保存标书正文。
如果输出文件已经存在，程序会要求更换文件名，不会覆盖已有成果。

## 构建 Windows 程序

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build.ps1
```

真实 Word 集成测试可单独执行：

```powershell
python -m pytest -m word_integration -v
```

构建成功后程序位于：

```text
dist/TenderWordFormatter/TenderWordFormatter.exe
```

## 常见问题

- **无法打开文件**：确认文件扩展名为 `.docx`，未加密且未被其他程序锁定。
- **目录未更新**：确认电脑安装了 Microsoft Word，并允许本程序启动 Word 自动化实例。
- **仍有未确认段落**：回到检查页面，处理所有“复核”或“高风险”条目。
- **封面字段缺值**：检查样板中的 `{{字段名}}`，并为每个字段提供值。
- **自动化失败**：源文件不会被修改；检查报告和错误提示后重新处理。

## 数据安全

文档分析阶段只读；所有修改都发生在独立输出副本。本程序只关闭自己创建的隐藏 Word 实例，不关闭用户已经打开的 Word。
